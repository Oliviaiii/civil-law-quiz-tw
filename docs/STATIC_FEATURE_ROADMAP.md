本文件集中追蹤書記官法科研習室的功能現況與後續優化。每次完成功能、調整優先順序或新增需求時，都必須同步更新勾選狀態、驗收項目與最後盤點日期。

## 實作原則

本站以 **GitHub Pages 可部署的純靜態網站**為優先，目前沒有後端服務。

- 功能必須能由靜態 HTML、CSS、JavaScript、React state、瀏覽器 Web API 與靜態 JSON 完成。
- 個人進度、收藏、模擬考紀錄及偏好設定，優先使用 `localStorage`；資料量明顯增加時才考慮 IndexedDB。
- 可使用 URL query/hash 保存題目深連結與可分享的篩選條件，但不能依賴伺服器動態路由。
- 離線功能可使用 Web App Manifest、Service Worker 與 Cache API。
- 不得為單一功能引入登入、會員、資料庫、Server Action、API Route、雲端 Queue 或常駐伺服器。
- 本機資料格式升級時，必須提供舊版 migration，並保持 JSON 匯出／匯入相容。
- 每項功能完成前，必須通過 GitHub Pages 靜態建置、測試、ESLint 與手機版操作檢查。
- 注意效能預算：首頁首次載入的 JavaScript 不應包含全部題庫；新增資料時留意 bundle 體積變化。
